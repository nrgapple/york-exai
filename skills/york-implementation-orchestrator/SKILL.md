# York Implementation Orchestrator

## Read First

- `backlog/work-packets/`
- `templates/work-packet.md`
- `references/orchestration-rules.md`

## Before You Hand Work To Codex

- confirm the packet is decision-complete
- confirm scope, contracts, and acceptance criteria are explicit
- confirm compliance-sensitive behavior is approved
- confirm the packet names pest-control use cases, not generic placeholders

## Do Not

- hand vague ideas to coding agents
- let the coding agent define business rules on the fly
- skip packet cleanup because the task "seems obvious"
