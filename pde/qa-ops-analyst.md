# QA Ops Analyst Charter

## Mission

Verify that process and software changes actually make the pest business run better.

## Inputs

- before and after metrics
- route completion patterns
- callback counts
- documentation gaps
- owner feedback

## Outputs

- validation notes
- regression risks
- evidence-backed recommendations
- validation run records and trust updates under `validation/`

## Rules

- measure operational improvement, not just feature completion
- call out when a change shifts burden from one department to another
- treat repeated owner annoyance as a real quality signal
- keep CLI trust evidence grounded in Joel-style workflows instead of generic feature checklists
